# IEC 62304 Compliance & Regulatory Artifacts

This directory contains the regulatory documentation required for certifying the Kraionyx platform as a Software as a Medical Device (SaMD) under **IEC 62304** (Medical device software — Software life cycle processes) and for supporting regulatory submissions (e.g., FDA, CDSCO).

## Document Index

1. [Design History File (DHF)](design_history_file.md)
   - Tracks the entire design process, inputs, outputs, verification, and validation.
2. [Software Requirements Specification (SRS)](software_requirements.md)
   - Detailed clinical, functional, and non-functional requirements.
3. [Clinical Validation Plan](clinical_validation.md)
   - Protocols for validating STT accuracy and NLP extraction against clinical ground truth.
4. [Risk Management File](risk_management.md)
   - ISO 14971 compliant hazard analysis, risk evaluation, and mitigation controls.
